# Preface

欢迎阅读 *Game Engine Architecture*。本书旨在完整讨论构成典型商业游戏引擎的主要组成部分。游戏编程是一个极其庞大的主题，因此我们需要覆盖大量内容。尽管如此，我相信本书讨论的深度足以让你扎实理解我们将要介绍的各个工程领域中的理论基础与常见实践。也就是说，本书实际上只是一次精彩且可能伴随终身的学习旅程的起点。关于游戏技术的各个方面，都有大量资料可供进一步学习，而本书既可以作为基础铺垫，也可以作为继续深入探索的跳板。

本书的重点将放在游戏引擎技术与架构上。这意味着我们会介绍构成商业游戏引擎的各个子系统背后的理论、通常用于实现这些子系统的数据结构、算法与软件接口，以及这些子系统如何在一个完整的游戏引擎中协同工作。游戏引擎与游戏本身之间的边界其实相当模糊。我们将主要关注引擎本身，包括大量底层基础系统、渲染引擎、碰撞系统、物理模拟、角色动画、音频系统，以及我称之为“游戏玩法基础层”（gameplay foundation layer）的深入讨论。这一层包括游戏对象模型、世界编辑器、事件系统和脚本系统。我们也会涉及游戏玩法编程的某些方面，包括玩家机制、摄像机和 AI。不过，由于范围所限，这些讨论主要会集中在游戏玩法系统与引擎之间的接口方式上。

本书可作为大学中级游戏编程两到三门课程系列的教材。它也适合业余软件工程师、游戏开发爱好者、自学游戏程序员，以及游戏行业中的从业者使用。初级工程师可以通过本书巩固自己对游戏数学、引擎架构和游戏技术的理解。而一些已经将职业发展集中于某一特定专业方向的资深工程师，也可以从本书所呈现的整体视角中获益。

为了充分利用本书，你应当具备基本面向对象编程概念的工作知识，并且至少具有一些 C++ 编程经验。游戏行业日常会使用多种编程语言，但工业级 3D 游戏引擎仍然主要使用 C++ 编写。因此，任何严肃的游戏程序员都需要能够用 C++ 编写代码。我们会在第 3 章中回顾面向对象编程的基本原则，并且你在阅读本书时无疑也会学到一些新的 C++ 技巧；不过，扎实的 C++ 基础最好还是通过参考文献 [51]、[52]、[40] 和 [41] 中的书籍来建立。如果你的 C++ 有些生疏，我建议你在阅读本书时参考这些书或类似书籍来复习相关知识。如果你此前没有任何 C++ 经验，那么在深入阅读本书之前，最好先至少阅读其中一本书的前几章，或者完成一些在线 C++ 教程。

学习任何类型的计算机编程，最好的方式都是亲自写代码。在阅读本书的过程中，我强烈建议你选择一些自己特别感兴趣的主题领域，并在这些领域中构思一些自己的小项目。例如，如果你对角色动画感兴趣，可以先安装 OGRE，并探索其中的蒙皮动画演示。随后，你可以尝试使用 OGRE 实现本书中介绍的一些动画混合技术。接下来，你也许可以实现一个简单的摇杆控制动画角色，让它能够在平面上四处移动。一旦你完成了一个相对简单、可以运行的东西，就在它的基础上继续扩展！然后再转向游戏技术的另一个领域。如此反复。项目具体是什么并不特别重要，重要的是你在练习游戏编程这门技艺，而不仅仅是在阅读关于它的内容。

游戏技术是一种不断发展、充满生命力的事物，永远无法被一本书的页面完全涵盖。因此，本书网站 [59] 会不定期发布额外资源、勘误、更新、示例代码和项目创意。你也可以通过 LinkedIn [60] 与我联系。

## New to the Fourth Edition

在 *Game Engine Architecture* 第三版出版到现在第四版出版之间，实时渲染与光照技术在复杂程度以及实现照片级真实感的能力方面都取得了巨大发展。在这一时期，GPU 的原始算力、特性集合和体系结构也经历了深刻变化。如今的 GPU 配备了全新的硬件组件，包括大量新的渲染特性与着色器类型、实时光线追踪加速器，以及用于机器学习的高速张量计算核心。为了回应这些变化，我意识到第四版 *Game Engine Architecture* 需要以一种全新的方式来讨论游戏引擎语境下的 3D 渲染与照片级真实光照。因此，本书现在专门增加了两章全新内容，用于讨论这些深刻而迷人的主题。

由于本书新增了如此多的内容，它的体量和重量也从一版到一版不断增加。一些读者曾告诉我，他们手中的这本书还有一个额外用途——可以放在家里的健身房里当作负重训练器材！为了避免本书在物理上变得过于笨重，第四版被拆分为两卷。*Volume I, Foundations and Core Engine Systems* 涵盖了构成每一个游戏引擎核心的基本概念、技术、工具和引擎系统。*Volume II, Graphics, Motion, and Sound* 则深入介绍每个游戏引擎中的主要子系统，包括动画引擎、3D 渲染引擎、光照引擎、物理引擎、音频引擎和游戏对象模型。虽然每一卷都可以单独阅读，但我强烈建议你获取并阅读两卷内容，这样才能对游戏引擎的整体图景形成完整认识。

第四版 *Game Engine Architecture* 也改进了以往版本中涉及的多个主题。书中新增了关于编译器优化语境下未定义行为的讨论；对 C++ 各语言标准的覆盖范围也扩展到了 C++23；动画章节新增了关于运动匹配的讨论。同时，和所有前几版一样，各种勘误也根据你们这些热心读者反馈给我的问题进行了修正。谢谢你们！我希望你们会发现，自己发现的错误现在都已经被修复了。（当然，它们很可能已经被一批新的错误取代了；如果你发现这些新错误，也欢迎随时告诉我！）

当然，正如我此前所说，游戏引擎编程领域几乎宽广和深邃到难以想象。一本书不可能覆盖所有主题。因此，本书的主要目的仍然是作为一种建立认知的工具，以及进一步学习的起点。希望这个版本能在你探索游戏引擎架构这一迷人且多面的领域时，对你的旅程有所帮助。
