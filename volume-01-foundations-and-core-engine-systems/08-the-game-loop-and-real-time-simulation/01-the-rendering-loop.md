## 8.1 渲染循环

在 Windows PC 或 Macintosh 上常见的那类图形用户界面（graphical user interface, GUI）中，屏幕上的大部分内容都是静态的。在任意给定时刻，任何一个窗口中只有一小部分内容会主动改变外观。正因为如此，图形用户界面传统上会通过一种称为**矩形失效**（rectangle invalidation）的技术绘制到屏幕上：只有屏幕中内容实际发生变化的小区域会被重新绘制。较早的 2D 电子游戏也使用类似技术，以尽量减少需要绘制的像素数量。

实时 3D 计算机图形的实现方式完全不同。当摄像机在 3D 场景中移动时，屏幕或窗口中的**全部内容**都会持续变化，因此“失效矩形”的概念不再适用。相反，运动和交互性的幻觉是以与电影非常相似的方式产生的：快速连续地向观看者呈现一系列静态图像。

显然，要在屏幕上快速连续地产生一系列静态图像，就需要一个循环。在实时渲染应用程序中，这有时称为**渲染循环**（render loop）。最简单的渲染循环结构如下：

    while (!quit)
    {
        // 根据交互输入，或通过沿着预定义路径移动，
        // 更新摄像机变换。
        updateCamera();

        // 更新场景中所有动态元素的位置、朝向，
        // 以及任何其他相关视觉状态。
        updateSceneElements();

        // 将一帧静态画面渲染到一个离屏帧缓冲区中，
        // 这个缓冲区称为“后缓冲区”。
        renderScene();

        // 将后缓冲区与前缓冲区交换，使最近渲染出的图像
        // 显示到屏幕上。
        // （或者，在窗口模式下，将后缓冲区的内容复制
        // （blit）到前缓冲区。）
        swapBuffers();
    }
