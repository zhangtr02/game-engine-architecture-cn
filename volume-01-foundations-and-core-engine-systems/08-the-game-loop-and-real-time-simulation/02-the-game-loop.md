# 8.2 游戏循环

游戏由许多相互作用的子系统组成，包括设备 I/O、渲染、动画、碰撞检测与碰撞解算、可选的刚体动力学模拟、多人网络、音频，等等。大多数游戏引擎子系统在游戏运行时都需要周期性地进行**服务**（servicing）。然而，不同子系统需要被服务的**频率**各不相同。动画通常需要以 30 Hz 或 60 Hz 的频率更新，并与渲染子系统同步。然而，动力学（物理）模拟实际上可能需要更高频率的更新（例如 120 Hz）。像 AI 这样的更高层系统可能每秒只需要服务一两次，而且它们未必需要与渲染循环同步。

实现游戏引擎子系统周期性更新的方式有很多种。稍后我们会探讨其中一些可能的架构。不过暂时先采用最简单的方式来更新引擎子系统：使用一个单一循环来更新所有内容。这样的循环通常称为**游戏循环**（game loop），因为它是服务引擎中每个子系统的主循环。

## 8.2.1 一个简单示例：Pong

Pong 是一种著名的乒乓球电子游戏类型，其起源可以追溯到 1958 年。当时，William A. Higinbotham 在布鲁克海文国家实验室创建了一款名为 *Tennis for Two* 的模拟计算机游戏，并在示波器上显示。这个类型后来因其在数字计算机上的后续版本而广为人知，其中包括 Magnavox Odyssey 游戏 *Table Tennis*，以及 Atari 街机游戏 *Pong*。

在 Pong 中，一个球会在两个可移动的垂直球拍和两面固定水平墙之间来回反弹。人类玩家通过控制旋钮来控制球拍的位置。（现代重新实现版本允许通过摇杆、键盘或其他人机接口设备进行控制。）如果球越过某个球拍而没有击中它，则另一方得分，球会被重置以开始新一轮游戏。

下面的伪代码展示了一个 Pong 游戏的核心游戏循环可能是什么样子：

    int main(int, char**) // Pong
    {
        initGame();

        while (true) // 游戏循环
        {
            readHumanInterfaceDevices();

            if (quitButtonPressed())
            {
                break; // 退出游戏循环
            }

            movePaddles();

            moveBall();

            collideAndBounceBall();

            if (ballImpactedSide(LEFT_PLAYER))
            {
                incrementScore(RIGHT_PLAYER);
                resetBall();
            }
            else if (ballImpactedSide(RIGHT_PLAYER))
            {
                incrementScore(LEFT_PLAYER);
                resetBall();
            }

            renderPlayfield();
        }

        return 0;
    }

显然，这个例子有些刻意。最初的 Pong 游戏当然不是通过以每秒 30 帧的速度重绘整个屏幕来实现的。当时 CPU 非常慢，几乎只能勉强实时绘制出表示球拍的两条线和表示球的一个方框。专用的 2D 精灵硬件经常用于在屏幕上绘制移动物体。不过，我们在这里关心的只是概念，而不是原版 Pong 的实现细节。

如你所见，当游戏首次运行时，它会调用 `initGame()`，执行图形系统、人机 I/O 设备、音频系统等可能需要的初始化设置。然后进入主游戏循环。语句 `while (true)` 表示这个循环会永远持续，除非它在内部被中断。循环中做的第一件事是读取人机接口设备。我们检查是否有任一人类玩家按下了“退出”按钮；如果按下了，就通过 `break` 语句退出游戏。接下来，在 `movePaddles()` 中，根据控制旋钮、摇杆或其他 I/O 设备当前的偏转量，稍微向上或向下调整球拍的位置。函数 `moveBall()` 将球当前的速度向量加到球的位置上，以得到下一帧的新位置。在 `collideAndBounceBall()` 中，随后会检查这个位置是否与两面固定水平墙以及球拍发生碰撞。如果检测到碰撞，则会重新计算球的位置，以考虑任何反弹。我们还会记录球是否撞到屏幕左边缘或右边缘。这意味着它错过了其中一个球拍；在这种情况下，我们会增加另一名玩家的得分，并将球重置到下一回合。最后，`renderPlayfield()` 绘制屏幕的全部内容。
