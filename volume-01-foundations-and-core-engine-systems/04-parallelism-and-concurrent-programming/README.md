# 4 并行与并发编程

计算性能——通常以每秒百万条指令（millions of instructions per second，MIPS）或每秒浮点运算次数（floating-point operations per second，FLOPS）来衡量——在过去四十年中一直以惊人的速度、并且相当稳定地提升。20 世纪 70 年代末，Intel 8087 浮点协处理器大约只能达到 50 kFLOPS（5 × 10⁴ FLOPS），而在大致同一时期，一台体积接近大型冰箱的 Cray-1 超级计算机可以达到约 160 MFLOPS（1.6 × 10⁸ FLOPS）。如今，PlayStation 5 或 Xbox Series X 等游戏主机中的 CPU 大约可以提供 10–12 TFLOPS（10¹³ FLOPS）的处理能力；而目前最快的超级计算机——美国的 Frontier 超级计算机——其 LINPACK 基准测试成绩为 1.102 EFLOPS（exa-FLOPS），也就是惊人的每秒 1.102 × 10¹⁸ 次浮点运算。对于个人计算设备而言，这意味着九个数量级的提升；对于超级计算机而言，则是十个数量级的提升。

许多因素共同促成了这种快速提升。早期，从真空管转向固态晶体管，使计算硬件得以小型化。随着新型晶体管、新型数字逻辑、新型基底材料和新制造工艺的发展，能够蚀刻到单个芯片上的晶体管数量大幅增加。这些进步也带来了功耗方面的改善，以及 CPU 时钟速度的大幅提升。并且，从 20 世纪 90 年代开始，计算机硬件制造商越来越多地转向 parallelism（并行性），将其作为提升计算性能的一种手段。

编写能够在并行计算硬件上 correctly（正确地）并 efficiently（高效地）运行的软件，要比为过去的串行计算机编写软件困难得多。它要求程序员深入理解硬件实际如何工作。更重要的是，要充分利用现代计算平台中的多核 CPU，需要采用一种称为 concurrent programming（并发编程）的软件设计方法。在并发软件系统中，多个控制流会协同工作，共同解决一个问题。这些控制流必须被仔细协调。许多在串行程序中运行良好的技术，一旦应用到并发程序中就会失效。因此，现代程序员（所有行业都如此，包括游戏行业）必须扎实理解并行计算硬件，并熟练掌握并发编程技术。
