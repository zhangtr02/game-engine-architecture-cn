# 16 游戏玩法系统导论

到目前为止，本书讨论的所有内容都集中在技术层面。我们已经了解到，游戏引擎是一个复杂的分层软件系统，它构建在目标机器的硬件、驱动程序和操作系统之上。我们已经看到，底层引擎系统如何为引擎其余部分提供所需服务；手柄、键盘、鼠标以及其他人机接口设备如何让玩家向引擎提供输入；渲染引擎如何在屏幕上生成 3D 图像；动画系统如何让角色和物体以自然方式运动；碰撞系统如何检测并解决形状之间的相互穿透；物理模拟如何使物体以符合物理规律的方式运动；3D 音频引擎又如何为我们的游戏世界渲染出可信且具有沉浸感的声景。然而，尽管这些组件提供了范围广泛而强大的功能，即使把它们全部组合在一起，我们仍然还没有得到一个游戏！

游戏并不是由技术定义的，而是由**玩法**（gameplay）定义的。玩法可以被定义为玩一款游戏时的整体体验。**游戏机制**（game mechanics）这个术语更具体地界定了这一概念——它通常被定义为支配游戏中各种实体之间交互的**规则**集合。它还定义了玩家的**目标**（objectives）、成功与失败的**判据**（criteria）、玩家角色的**能力**（abilities）、游戏虚拟世界中存在的**非玩家实体**（non-player entities）的数量与类型，以及整个游戏体验的**整体流程**（overall flow）。在许多游戏中，这些元素会与引人入胜的故事以及丰富的角色阵容交织在一起。不过，并非每一款电子游戏都一定需要故事和角色，像 *Tetris* 这样极为成功的益智游戏就是明证。在论文 “A Survey of ‘Game’ Portability” [382] 中，谢菲尔德大学的 Ahmed BinSubaih、Steve Maddock 和 Daniela Romano 将用于实现游戏玩法的一组软件系统称为游戏的 **G-factor**。在接下来的三章中，我们将探讨那些定义并管理一款游戏的游戏机制（也即 gameplay，亦即 G-factor）的关键工具和引擎系统。
