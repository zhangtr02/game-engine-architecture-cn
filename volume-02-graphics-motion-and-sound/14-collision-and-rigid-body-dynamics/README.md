# 14 碰撞与刚体动力学

在真实世界中，固体对象天生就是——嗯——**固体**。它们通常会自行避免做那些不可能的事情，比如彼此穿透。但在虚拟游戏世界中，对象不会主动做任何事情，除非我们告诉它们去做；游戏程序员必须显式地确保对象不会彼此穿透。这正是任何游戏引擎中一个核心组件的职责——**碰撞检测系统**（collision detection system）。

游戏引擎的碰撞系统通常与**物理引擎**（physics engine）紧密集成。当然，物理学是一个非常广阔的领域，而当今大多数游戏引擎所说的“物理”（physics），更准确地说，是一种**刚体动力学仿真**（rigid body dynamics simulation）。**刚体**（rigid body）是一种理想化的、无限坚硬且不可变形的固体对象。术语**动力学**（dynamics）指的是确定这些刚体在**力**（forces）的影响下如何随时间**运动**并**相互作用**的过程。刚体动力学仿真允许以一种高度交互且自然混沌的方式，把运动赋予游戏中的对象——如果使用预制动画片段来移动物体，这种效果就要难实现得多。

动力学仿真会大量使用碰撞检测系统，以便正确模拟仿真中对象的各种物理行为，包括彼此反弹、在摩擦力作用下滑动、滚动，以及最终静止。当然，碰撞检测系统也可以脱离动力学仿真单独使用——许多游戏根本没有“物理”系统。但所有涉及对象在二维或三维空间中运动的游戏，都具有某种形式的碰撞检测。

在本章中，我们将研究典型碰撞检测系统和典型物理（刚体动力学）系统的架构。在考察这两个紧密相关的系统的组成部分时，我们也会了解支撑它们的数学与理论。
