# 11 渲染

当大多数人想到计算机游戏时，首先浮现在脑海中的往往是令人惊叹的三维图形。如今，大多数游戏都致力于生成**照片级真实感图形**（photorealistic graphics）——也就是细节丰富、光照效果在物理上足够精确的图像，以至于有时很难分辨游戏中的虚拟世界与现实之间的差别。有些游戏则选择更具风格化的视觉表现，也许会模拟卡通、铅笔素描，或印象派绘画的外观。当然，二维游戏图形在游戏产业中仍然占有非常重要的地位。在接下来的两章中，我们将主要关注实时三维图形，并且通常假设目标是实现照片级真实感渲染。不过，我们将要学习的几乎所有内容，也同样可以应用于非照片级真实感的 3D 渲染；同时，我们也会简要介绍一些实现风格化效果的方法。

3D 计算机图形学这一主题，是一个更宽泛领域的子集，这个更宽泛的领域被称为**计算机生成图像**（computer-generated imagery, CGI）。计算机图形技术在游戏产业之外也有许多用途，包括电影与特效、仿真、计算机辅助设计（computer-aided design, CAD）、照明工程、医学可视化，以及更多领域。一般来说，CGI 的目标是以某种非图像数据作为输入，并输出一幅合成图像。

**实时 3D 渲染**（real-time 3D rendering）是 3D 渲染中的一种特殊类型，它旨在以交互式帧率生成达到期望质量和照片级真实感水平的图像，从而营造运动的幻觉。这意味着刷新率至少要达到每秒 30 帧（frames per second, FPS），不过在游戏产业中，60 FPS 或更高的帧率更为常见。在 60 FPS 的帧率下，计算机只有略多于 16 毫秒的时间来生成每一幅图像。但游戏引擎在每一帧中所做的事情远不只是渲染图形；它还需要更新游戏逻辑、为场景中的对象播放动画、执行碰撞检测和物理仿真、合成音频，等等。因此，一个游戏的渲染引擎需要同时满足两个同等重要的要求：一方面要生成高度细致的照片级真实感图形，另一方面还必须以非常、非常快的速度完成这些工作！考虑到电影渲染引擎通常要花费数分钟乃至数小时才能渲染出单帧图像，如今实时计算机图形所达到的质量确实令人惊叹。

实时 3D 渲染是一个极其宽广而深刻的主题，因此我们不可能在两章之内覆盖所有细节。幸运的是，关于这一主题已经有许多优秀的书籍和其他资源。事实上，实时 3D 图形也许是构成游戏引擎的所有技术中资料最为丰富的领域之一。因此，本章与下一章的目标，是为你提供一个关于渲染技术的整体理解，并作为进一步学习的起点。读完这些内容之后，你再去阅读其他 3D 图形学书籍时，应该会发现那更像是一段穿行于熟悉领域的旅程。
