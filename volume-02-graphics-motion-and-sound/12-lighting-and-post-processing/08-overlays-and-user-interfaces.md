## 12.8 叠加层与用户界面

每个游戏至少都需要某种**图形用户界面**（graphical user interface，简称 GUI 或 UI）。UI 元素包括**平视显示器**（heads-up display，HUD）、武器选择界面，以及各种菜单。连同 Section 10.2 中讨论过的调试绘制设施，这些视觉元素有时也被称为**叠加层**（overlays），因为它们通常是在主 3D 场景渲染完成之后再绘制的。

叠加层通常会在禁用 z 测试的情况下渲染，以确保它们显示在三维场景之上。二维叠加层通常通过在屏幕空间中使用正交投影渲染四边形（三角形对）来实现。三维叠加层可以使用正交投影渲染，也可以通过常规透视投影渲染：此时几何体被放置在视图空间中，使其随摄像机一起移动。

用户界面叠加层通常由一组二维图形图元组成。这些图元大体可以分为三类：

- **文本**（text）。每个用户界面都需要向用户显示人类可读的字符串。因此，游戏的用户界面系统必须能够渲染文本，通常还要支持多种字体样式和字号。

- **位图图像**（bitmapped images）。图标在游戏用户界面中非常常见，几乎覆盖了 UI 中所有非文本的视觉元素。它的范围很广，从滚动条或下拉组合框两端的小箭头，到表示玩家角色当前持有武器的图标，再到提示玩家与游戏中物品交互的悬浮三角按钮图标。用户界面还经常需要显示较大的图像，例如截图或角色肖像。

- **嵌入式视频**（embedded video）。许多用户界面支持在屏幕上的某个视口中播放预渲染影片。

- **矢量图形**（vector graphics）。以数学线段和曲线表示，并在运行时动态光栅化的视觉元素称为**矢量图形**元素。矢量图形可用于绘制细线、方框、圆、弧线等。图标也可以用矢量图形来实现，而不是使用位图。**可缩放矢量图形**（Scalable Vector Graphics，SVG）数据格式是一种用于编码和光栅化矢量图形的标准化方式。

### 12.8.1 文本与字体

游戏引擎的文本/字体系统通常被实现为一种特殊的二维（有时也可以是三维）叠加层。文本渲染系统的核心能力，是在屏幕上以各种方向显示一串字符字形，这些字形对应于某个文本字符串。

字体通常通过一种称为**字形图集**（glyph atlas）的纹理贴图来实现，其中包含各种所需的字形。该纹理通常只包含一个 alpha 通道——每个像素的值表示该像素被字形内部覆盖的比例。字体描述文件会提供一些信息，例如每个字形在纹理中的包围盒，以及字距调整、基线偏移等字体布局信息。渲染一个字形时，会绘制一个四边形，其 `(u, v)` 坐标对应于图集纹理贴图中目标字形的包围盒。

纹理贴图提供 alpha 值，而颜色则单独指定，因此可以从同一个图集中渲染任意颜色的字形。

字体渲染的另一种选择，是使用 FreeType [296] 这样的字体库。FreeType 库允许游戏或其他应用程序读取多种格式的字体，包括 TrueType（TTF）和 OpenType（OTF），并将字形渲染为任意所需点大小的内存位图。FreeType 使用字形的贝塞尔曲线轮廓来渲染每个字形，因此可以产生非常精确的结果。

通常，像游戏这样的实时应用会使用 FreeType 将所需字形预渲染到一个图集中，然后再把这个图集作为纹理贴图使用，每一帧将字形渲染为简单四边形。不过，如果在引擎中嵌入 FreeType 或类似字体库，也可以按需动态地将某些字形渲染到图集中。这在渲染中文或韩文这类拥有大量可能字形的语言文本时会很有用。

另一种渲染高质量字符字形的方法，是使用**有符号距离场**（signed distance fields）来描述字形。在这种方法中，字形仍然被渲染为像素图，但每个像素的值不再是 alpha “覆盖率”值。相反，每个像素包含一个有符号距离，表示该像素中心到字形最近边缘的距离。在字形内部，距离为负；在字形轮廓外部，距离为正。当从有符号距离场纹理图集中渲染字形时，这些距离会在像素着色器中用于计算非常精确的 alpha 值。最终结果是文本在任意距离和视角下都显得平滑。FreeType 的最新版本支持将字形光栅化为有符号距离场。关于有符号距离场文本渲染，可以在线搜索 Konstantin Käfer 的文章 “Drawing Text with Signed Distance Fields in Mapbox GL”，或 Valve 的 Chris Green 所写的文章 “Improved Alpha-Tested Magnification for Vector Textures and Special Effects”。

字形也可以直接从定义它们的贝塞尔曲线轮廓进行渲染。Terathon Software LLC 的 Slug 字体渲染库在 GPU 上执行基于轮廓的字形渲染，从而使这种技术在实时游戏应用中变得可行。

一个良好的文本/字体系统必须考虑不同语言固有的字符集差异和阅读方向差异。将文本字符串中的字符排布出来的过程称为对字符串进行**塑形**（shaping）。根据语言不同，字符可以从左到右或从右到左排布，并且每个字符都会对齐到共同的**基线**（baseline）上。字符之间的间距一部分由字体创建者提供的**度量信息**（metrics）决定（这些信息存储在字体文件中），另一部分由**字距调整**（kerning）规则决定，这些规则会根据上下文调整字符之间的间距。

一些文本系统还提供各种有趣的功能，例如让字符以不同方式在屏幕上移动的动画能力、单独为每个字符设置动画的能力等等。不过，在实现游戏字体系统时，重要的是要记住：只应实现游戏**实际需要**的功能。例如，如果游戏从来不需要显示动画文本，那么为引擎配备高级文本动画功能就没有意义。
